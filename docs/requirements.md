## 1. プロジェクトの目的

学習データを構造化し、多角的な分析（ビジュアル・アナリティクス）によって、**合格への最短ルートを特定する**。

---

## 2. 画面構成

### ① エントリ・ゲート（ログイン）

**機能**: 認証および操作対象試験の切り替え。

| 項目 | 詳細 |
|------|------|
| ユーザーID | テキスト入力 |
| パスワード | テキスト入力（マスク表示） |
| 対象試験 | 作成済みリストからプルダウン選択 |

---

### ② ターゲット・レジストリ（試験登録・設定）

**機能**: 学習対象の定義とロードマップの構築。

| 項目 | 詳細 |
|------|------|
| 試験名称 | 自由入力（例：基本情報技術者試験、AWS SAP など） |
| 出題範囲設定 | 「あり・なし」を選択 |
| 階層デザイン | 「あり」の場合、親カテゴリに子をぶら下げる再帰構造（階層数自由）を定義。「＋」ボタンで動的に追加 |
| 試験日設定 | 本番試験日のカレンダー選択 |

#### 階層デザインのイメージ

```
試験名（例：基本情報技術者試験）
├── テクノロジ系
│   ├── コンピュータ構成要素
│   │   ├── プロセッサ
│   │   └── メモリ
│   └── システム構成要素
└── マネジメント系
    └── プロジェクトマネジメント
```

## 4. ロジック仕様

### 階層管理

`ExamID > ParentID > CategoryID` の構造で管理する。
異なる試験間・同一試験内の同名カテゴリ競合を完全に回避する。

```
ExamID
└── ParentID（親カテゴリ）
    └── CategoryID（子カテゴリ）
```

### 比較ロジック

学習データは以下の3時点で比較する。

| 比較軸 | 定義 |
|--------|------|
| 初回（Initial） | 該当試験の最古レコード |
| 前回（Previous） | 最新から数えて1つ前のレコード |
| 最新（Latest） | 直近の入力レコード |

### データ保存方針

- 入力された全履歴を**蓄積型**で保存する
- 上書きは行わず、レコードを追記していく
- 蓄積データを元に週次推移を算出する
